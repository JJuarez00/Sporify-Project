1. Project Overview

dawnFM* is a personal Spotify stats dashboard for music listeners who want to see their listening habits outside of Spotify Wrapped. After signing in with Spotify, users can view their top tracks and top artists across Spotify's supported time ranges: short_term, medium_term, and long_term.

The app is designed for users like Joey from the original proposal: someone who listens to music constantly and wants an easy way to show their current music taste. The final version focuses on a working dashboard, artist search, track result modals, 30 second previews when available, light/dark mode, and a PDF export of the user's current stats.

5. Challenges and Decisions

The final version changed from the original proposal because Spotify's API and policy limits affected what data the app could safely and reliably show.

  • Daily stats were removed because Spotify's top-items endpoint only supports short_term, medium_term, and long_term ranges.
  • Public artist data is not cached in the final project. Spotify's terms limit local caching of Spotify content to what is strictly necessary and temporary, so the app fetches artist and track data live instead.
  • The app stores only the logged-in user's profile/session data needed for the dashboard, and that data is removed when the user signs out.
  • Several Spotify values changed or were deprecated, including Get Artist Top Tracks, track preview_url, and the popularity field. Because of this, the artist modal uses Spotify Search for track results and spotify-preview-finder to locate preview clips when available.
  • The UI also had challenges, especially building a shared navigation bar that worked across every page while still showing the correct login state, profile image, display name, and dashboard link.
  • The responsive hamburger menu was another design decision because the desktop navigation did not fit well on smaller screens. GSAP was used to animate the menu opening, closing, and icon movement so the mobile experience felt intentional.
