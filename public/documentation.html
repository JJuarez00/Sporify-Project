Technical Documentation

N320 — Intermediate Media Application Development — Final Project

1. Project Overview

dawnFM* is a Spotify stats dashboard built as a final project for N320. It gives users a way to see their listening habits at any time, not just once a year through Spotify Wrapped. After signing in with a Spotify account, users can view their top tracks and top artists across three time ranges: the last 4 weeks, the last 6 months, and all time.

The search page is open to anyone without an account. It lets you look up any artist on Spotify, browse track results, and play 30-second audio previews where available.

The project was designed to be straightforward: sign in, see your stats, explore artists. No tracking beyond what Spotify already provides, and no data stored beyond what is needed to keep the session running.

3. Feature Walkthrough

  • Spotify Login — Users sign in through Spotify's OAuth flow. After authenticating, Spotify sends a temporary code back to the app, which gets exchanged for an access token and a refresh token. The access token is used to fetch personal data, and the refresh token is used to get a new one when it expires so the user does not have to sign in again.
  • Dashboard — The dashboard shows the user's top tracks and top artists side by side. Both sections have dropdowns to switch between time ranges and to control how many results are displayed. Tracks include a 30-second preview button when a clip is available.
  • Artist Search — The search page lets anyone look up an artist by name without signing in. Results come from Spotify Search and display as cards. Clicking a card opens a modal with track results for that artist, along with preview buttons where audio is available.
  • Audio Previews — Preview clips are sourced through the spotify-preview-finder package, since Spotify's native preview_url field was deprecated. Only one track plays at a time — starting a new preview automatically stops the previous one.
  • PDF Export — From the dashboard, users can download a PDF of their current top tracks and artists using the jsPDF library. The export reflects whatever time range and limit are currently selected.
  • Light / Dark Mode — A toggle button in the bottom right corner switches between light and dark mode. The preference is saved to localStorage so it persists across visits.

4. API Documentation

The app connects to the Spotify Web API using two access methods depending on what data is needed.

Client Credentials — no user login required

Used for public catalog data like search results and artist details. The server requests a token using the app's client ID and secret and caches it until it expires.

Endpoint What it's used for
POST /api/token Requests a public access token using client credentials
GET /v1/search?type=artist Searches for artists by name on the search page
GET /v1/artists/{id} Fetches artist details (name, image) for the modal header
GET /v1/search?type=track Gets track results for an artist — used as a replacement for the deprecated top-tracks endpoint

Authorization Code Flow — user login required

Used for personal listening data. The user signs in through Spotify and grants permission, and the app receives tokens scoped to that user's account.

Endpoint What it's used for
POST /api/token (code exchange) Exchanges the OAuth callback code for an access token and refresh token
POST /api/token (refresh) Gets a new access token when the old one expires, without requiring the user to log in again
GET /v1/me Fetches the logged-in user's display name, profile image, and Spotify ID
GET /v1/me/top/tracks Fetches the user's top tracks for the selected time range and limit
GET /v1/me/top/artists Fetches the user's top artists for the selected time range and limit

Time ranges

The top-items endpoints accept a time_range parameter with three possible values:

  • short_term — approximately the last 4 weeks
  • medium_term — approximately the last 6 months
  • long_term — calculated across all available listening history

5. Challenges and Decisions

The final version of this project changed from the original proposal because of Spotify API deprecations and policy limits that affected what the app could reliably show.

  • No daily stats — The original proposal planned for a daily time range, but Spotify's top-items endpoint only supports short_term, medium_term, and long_term. Daily breakdowns are not available through the API.
  • No caching of public artist data — Spotify's developer terms limit local caching of their content to what is strictly necessary. Public artist and track data is fetched live on each request rather than stored in the database.
  • Deprecated endpoints and fields — Spotify deprecated the Get Artist Top Tracks endpoint for dev-mode apps in February 2026, along with the native preview_url field on track objects. The app works around this by using Spotify Search for artist track results and the spotify-preview-finder package to locate audio preview clips.
  • Shared navigation state — The nav bar needed to show the correct login state, profile image, and dashboard link across every page. This was handled by injecting the nav from a shared nav.js component and running a session check on each page load to update the UI accordingly.
  • Mobile navigation — The desktop nav didn't fit well on smaller screens, so a hamburger menu was added for tablet and phone sizes. GSAP was used to animate the menu open and close, including the icon lines rotating into an X.
  • Parallel API requests — When loading preview URLs for multiple tracks, the original code fetched them one at a time in a loop, which was slow. This was replaced with Promise.all so all preview requests run at the same time.

6. How to Run

Follow these steps to run the project locally.

  1. Install dependencies 
    npm install
  2. Create a Spotify app — Go to developer.spotify.com/dashboard, create a new app, and add http://localhost:3000/callback as a redirect URI.
  3. Set up environment variables — Create a .env file in the project root: 
    SPOTIFY_CLIENT_ID=your_client_id
SPOTIFY_CLIENT_SECRET=your_client_secret
SPOTIFY_REDIRECT_URI=http://localhost:3000/callback
  4. Start the server 
    npm start
  5. Open the app — Visit http://localhost:3000 in a browser.
